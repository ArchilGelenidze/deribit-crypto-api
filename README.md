## Описание тестового задания
Написать клиент для криптобиржи Deribit (https://docs.deribit.com/).
Клиент должен каждую минуту забирать с биржи текущую цену btc_usd и eth_usd (index price валюты)
После чего сохранять в базу данных тикер валюты, текущую цену и время в UNIX timestamp.
Написать внешнее API для обработки сохраненных данных на FastAPI.

### Обязательные требования 

API должно включать в себя методы:
- получение всех сохраненных данных по указанной валюте
- получение последней цены валюты и получение цены валюты с фильтром по дате.
- Все методы должны быть GET и у каждого метода должен быть обязательный query-параметр ticker.
---
- В качестве БД использовать PostgreSQL.
- Код выложить на GitLab с подробным README и документацией по разворачиванию.
- В README добавить секцию Design decisions.
- Для периодического получения цен использовать Celery.

### Необязательные требования 
- Написать unit тесты для основных методов.
- Развернуть приложение в двух контейнерах: для приложения и базы данных.
- Применить aiohttp при написании клиента.

### Критерии оценки 
- Чистая архитектура и чистый код.
- Нейминг.
- Отсутствие глобальных переменных.
- Умение использовать ООП.
- Понимание принятых решений и способность их объяснить.

## Мое решение и описание
Асинхронный REST API сервис для периодического сбора и предоставления исторических цен криптовалют (BTC, ETH) с биржи Deribit. 


## 🛠 Технологический стек

* **Язык:** Python 3.12
* **API Фреймворк:** FastAPI
* **База данных:** PostgreSQL, SQLAlchemy (async), Alembic (миграции)
* **Фоновые задачи:** Celery, Redis (Broker & Backend)
* **Тестирование:** Pytest
* **Инфраструктура:** Docker, Docker Compose

## 🚀 Быстрый старт (Quick Start)

Запуск проекта полностью автоматизирован через Docker Compose.

1. Склонируйте репозиторий:
```bash
git clone https://github.com/ArchilGelenidze/deribit-crypto-api.git
```

2. Перейдите в нужную директорию:
```bash
cd deribit-crypto-api 
```

3. Создайте файл переменных окружения из шаблона:
``` bash
cp .env.example .env
```

4. Запустите оркестратор:
```bash
docker compose up --build
```

## 📡 API Эндпоинты

API предоставляет следующие методы (подробные схемы запросов и ответов доступны в Swagger):

- `GET /prices/all` — Получение истории цен с поддержкой пагинации (limit, offset) и фильтрацией по тикеру.

- `GET /prices/latest` — Получение самой свежей цены для выбранного тикера.

- `GET /prices/by-date` — Получение цены на конкретную дату и время (ISO 8601). Если точного совпадения нет, возвращается ближайшая предшествующая запись.

### ⚙️ Архитектурные решения (Design Decisions)

При разработке были приняты следующие инженерные решения:

- Разделение логики (Celery + FastAPI): Процесс парсинга данных с биржи вынесен в изолированные фоновые воркеры Celery. Это гарантирует, что задержки сети при обращении к внешнему API Deribit никак не повлияют на Event Loop веб-сервера.


- Безопасная пагинация: В эндпоинте получения истории цен реализована пагинация (Limit/Offset) с жестким ограничением максимального количества возвращаемых записей (не более 1000 за запрос). Это защищает базу данных от перегрузки и предотвращает утечки памяти на сервере.


- Строгая валидация входящих данных (Enums): Валидация поддерживаемых тикеров реализована на уровне роутинга через Enum. Это отсекает некорректные запросы до обращения к базе данных и улучшает автогенерацию документации Swagger.
 

- Healthchecks в Docker Compose: Реализована строгая очередность запуска контейнеров. API и Celery Worker стартуют только после того, как база данных PostgreSQL и Redis сообщают о полной готовности принимать соединения (pg_isready), что исключает ошибки старта.
